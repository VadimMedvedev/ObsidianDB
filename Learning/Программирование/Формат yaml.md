
Файлы такого формата часто используються для хранения конфигурационных данных. Они имеют разрешение .yml или .yaml.

### Синтаксис

Каждый уровень вложенности обозначаеться двумя пробелами, не tab, это очень важно!!!

Комментарии пишуться после символа #.

Базовая структура для файла, с которой всё начинаеться, являеться словарь.
Всего базовых структур есть несколько, это строки (которые можно писать без ", но можно и с ковычками), числа, булевые значения (true, false), словари и списки.

##### Словарь

``` yaml
me:
  name: Vadim
  isMathematic: true
  subscribers: 300
```
ключи записываються подряд, потом двоеточие, пробел (обязательно) и значение, коем может быть любой из выше перечисленных типов.

Короткий формат (не подойдёт при большой вложенности):
``` yaml
me: {name: "Vadim", isMathematic: true, subscribers: 300}
```

##### Список

``` yaml
skills:
  - "Programming"
  - "Math"
  - "Guitar"
```
здесь  обязателен отступ (два пробела), минус и пробел. Порядок следования в списке важен, в отличие от словаря.

Короткий формат (не подойдёт при большой вложенности):
``` yaml
skills: ["Programming", "Math", "Guitar"]
```

##### Вложенность

в списке можно перечеслять словари, или же в качестве значений словаря может быть список, всё что только душе угодно!!
