git init - инициализирует репозиторий
git status - выводит информацию об отслеживании файлов
git add <файл> - добавляет файл в список отслеживаемых файлов
git commit -m "комметарий" - сохраняет все изменения в репозитории
git log - показывает все коммиты

После каждого изменения файлов их надо добавлять в список изменеий (команда add)!!! и только потом коммитить. 

git remote add origin <ссылка на репозиторий в github> - добавляет удалённый репозиторий с сайта github. origin 

git branch -M main - переименовывает ветку в который мы сидим. -M сокращённые ключи --move --force.

git push -u origin main - отправляет последние комиты в удалённый репозиторий

в папке .git/ в локальном репозитории лежат следующие директории:
1) hooks/ - скрипты, которые реагируют на разные события
2) info/ - информация о репозитории
3) logs/ - логи
4) objects/ - хранит все данные репозитория захэшированные
5) ref/ - "закладки"
Что ещё храниться?:
COMMIT_EDITMSG -  git здесь хранит сообщения пользователя
config - хранит конфиги
description - описание репозиториев
HEAD - хранит в какой ветке вы сейчас сидите
index - хранит список файлов подготовленных к commit

ВАЖНО!!! Сами не менять все эти настройки и значение файлов (кроме конечно конфигов).
Папка .git не коммитеться.

Каждый коммит это по сути полный бэкап, к которому можно откатиться.
Каждый коммит хранит уникальный хэш, автора комита, сообщение, время и т.д.

Ключи git log:
1) --oneline - кратко показывает историю коммитов
2) --graph --all - граф коммитов из разных веток

Файл .gitignore содержит те файлы которые не надо сохранять в коммитах.

### Ветвление

Можно создавать параллельные ветки, чтобы отдельно работать над отдельными модулями и при успехе сливать в осовную ветку.

Основные команды

git branch <название> - создание ветки
git branch - просмотр всех веток
git branch -d <название> - удаление веток
git branch -m <старое название> <новое> - переименование ветки

git checkout <название> - переключение на другую ветку
git checkout -b <название> - переключение и автоматическое создание при этом
git checkout <название ветки> -- <путь до файла> - копирует файл из данной ветки в текущую

git stash - переключиться на другую ветку

git merge <ветка> - сливает данную ветку в ту в которую ты находишься

При merge могут возникнуть конфликты.

git merge --abort - отмена мёрджа, если не хочеться разрешать конфликты

### GitHub

Вкладки:
1) Code - показывает все файлы в основной ветке, все ветки
	* star - плейлисты репозиториев по темам нужным, как на ютубе
	* pin - закрепить
	* code - копирует ссылку на репозиторий
	* README - файл с инфой о проекте
	* В центре - все файлы из основной ветки, можно заходить в директории, заходить в файлы.  Скачивать файлы
	* fork - делает полную копию репозитория
	* commits - проссматривает историю всех коммитов 
2) issuse - вкладка о ошибках, можно отмечать что исправить
3) pull, request - проверка, дороботка и т.д. запрашивать апргрейды и т.д.
4) action - автоматическая проверка ошибок при помощи , по типу макросов, помогающие проверять ошибки
5) projects - инструмент над работой проектами (canban доски и т.д.)
6) wiki - документация, чтобы не забыть что делали
7) security - вкладка с сообщениями о безопасности. Можно назначать роли, кто что может делать
8) insights - статистика: коммиты пользователей, ветки и т.д.
9) settings - настройки проекта
	* colloborators - можно добавить людей в команду над репозитрорием
