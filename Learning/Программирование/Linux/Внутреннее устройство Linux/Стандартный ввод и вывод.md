Linux использует потоки ввода и вывода ***(I/O streams)*** для записи и чтения данных. 
Все [[процессы]] читают данные из потока ввода, и записывают данные в поток вывода. Источником потока ввода может быть файл, устройство, окно терминала, или даже поток вывода другого процесса.

Например, если написать в командной строке:
``` bash
cat
```
то [[Команда cat]], т.к. после неё нет файлов, будет использовать стандартный ввод, в данном случае это будет наш терминал. Если что-то написать и нажать ENTER, то cat тоже это напишет. Так будет продолжаться, пока вы не нажмёте CTRL-D (что заканчивает файл при помощи EOF (end-of-file)).

[[Ядро]] Linux само предоставляет процессам стандартный ввод и вывод. Так что в обычных условиях стандартным вводом и выводом будет терминал, из которого был запущен процесс.
Часто стандартные потоки ввода и вывода обозначают *stdin* и *stdout*.
Зачастую, если в программу не передают входной файл, то она читает из stdin. Выводят данные программы обычно в стандартный вывод, а иногда только в него.
